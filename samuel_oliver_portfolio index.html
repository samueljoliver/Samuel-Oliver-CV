The Traditional Split

In a traditional card payment split, the server must divide the total bill by the 4 diners, type in that amount, have the first diner tap their card, wait for authorization, wait for the receipt to print, and then repeat this process 4 times in a row. This is time consuming and slow.

  1. Calculate & Key Amount
  2. Customer Taps Card
  3. Wait for Authorization
  4. Print Receipt
  5. Repeat Loop for next payer...

Visa Bill Share Flow

Visa Bill Share allows the server to type in the total amount once, select the split and then: tap, tap, tap, tap. A single authorization and receipt is provided saving time and hassle.

  1. 1. One Touch Splitting
  2. 2. Stepless Credential Capture
  3. 3. Simultaneous Authorisation
  4. 4. Single Receipt Provision
